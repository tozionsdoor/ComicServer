1. 接続の仕組み

ArcHive は接続状況に応じて、以下の優先順位で自動的に最適な方法を選びます。

接続方法使われる場面速度
LAN直結 スマホとサーバーPCが同じWi-Fiにある 高速
IPv4外部接続 外出先 ＋ ルーターのポート開放が成功している 高速
IPv6直結 外出先 ＋ サーバー側にIPv6グローバルアドレスがある 高速
WebRTC（P2P） 上記がすべて使えない場合の最終手段 やや遅い
ポート自動開放について：「接続設定」の「外出先からのIPv4自動接続を許可（UPnP）」にチェックが入っていると、サーバー起動時にルーターのポートを自動で開放します。ログに UPnP: 外部ポート XXXXX→8765 を自動開放しました と出れば成功です。

2. はじめての使い方

  1. サーバー画面の「本棚登録」で本（マンガやPDF、他書庫等）が入っているフォルダを追加し「設定を保存」を押す。
  2. 「▶ サーバー起動」ボタンを押す。ステータスが「起動中」になることを確認。
  3. スマホアプリを起動し、「LAN内を探す」をタップする。
  4. 一覧にサーバー名が表示されたらタップ → サーバー側で「端末管理」を開いて承認する。
  5. 以降は外出先でも自動で接続方法を切り替えて接続されます。
注意：初回接続は必ず同じWi-Fi（自宅内）で行ってください。スマホがモバイルデータ通信中は「LAN内を探す」でサーバーが見つかりません。

3. サーバーログの読み方

サーバー画面のログ欄に表示されるメッセージの意味を解説します。

接続ログ（アプリがつながるたびに表示）

[HTTP] 接続: 192.168.1.10 (LAN)            ← 同じWi-Fi内からHTTP直結。最適な状態。
[HTTP] 接続: 49.105.103.1 (IPv4外部)       ← 外出先からIPv4で直結。良好な状態。
[HTTP] 接続: 2400:4050::1 (IPv6直結)      ← 外出先からIPv6で直結。良好な状態。
[WebRTC] DC接続: Android                  ← WebRTC経由。動くが速度が落ちる場合あり。

ポート開放ログ（サーバー起動時に表示）

[IPv4] UPnP: 外部ポート 52736→内部 8765 を自動開放しました  ← 外出先接続の準備OK
[IPv4] UPnP: 失敗（ルーター非対応またはUPnP無効）            ← 外出先HTTP不可 → WebRTCへ切替

接続情報更新ログ（外出先からの接続に使われる情報）

[Firebase] host を更新しました (ipv4=49.x.x.x:52736)     ← 外出先アプリへ接続先を通知
[Firebase] host は変更なし、更新スキップ                   ← 正常。変化がないため省略。
良好な状態の目安：起動後に UPnP: 外部ポート XXXXX→8765 を自動開放しました が出て、接続時のログに (LAN) または (IPv4外部) が表示されていれば最適な状態です。

4. つながらないときのチェックリスト

▶ ブラウザで本棚が開けない

  1. サーバー画面のステータスが「起動中」になっているか確認する。
  2. 「ブラウザで開く」ボタンを押してブラウザが開くか確認する。
  3. ブラウザに「プライバシーエラー」「証明書の警告」が出た場合 →「詳細設定」→「サイトへ進む（安全でない）」をクリック。
    （ArcHiveは自己署名証明書を使用しています。警告が出ますが安全なローカル接続です）
  4. それでも開けない場合 → Windowsファイアウォールでポート 8765 が許可されているか確認する。

▶ アプリからLAN接続できない

  1. スマホとサーバーPCが同じWi-Fiに接続していることを確認する。
  2. サーバーが「起動中」になっているか確認する。
  3. サーバーの「端末管理」を開いて「承認待ち」のスマホが届いていれば承認する。
  4. 上記で解決しない場合：
    • Windowsファイアウォールでポート 8765 がブロックされていないか確認する。
    • ウイルス対策ソフトが通信をブロックしていないか確認する。
    • PCが企業・学校ネットワークに接続している場合、ポートが制限されていることがある。

▶ 外出先から接続できない

まずサーバーのログを確認してください。

ログの内容状態対処
(IPv4外部) の接続ログが出ている 正常 接続できているはずです。アプリを再起動してみてください。
UPnP: 外部ポート XXX を自動開放しました 準備OK ポート開放成功。しばらく待つと外出先から接続できるはずです。
UPnP: 失敗 または表示なし 要確認 ルーターのUPnPが無効か非対応。→ 接続しにくい環境 を参照。
ログに [WebRTC] のみ表示 WebRTCのみ HTTP直結できない状態。速度が落ちる場合あり。→ 接続しにくい環境 を参照。

▶ 接続はできるが画像が遅い・途切れる

  • ログに [WebRTC] のみ → ポート開放できていない可能性。接続しにくい環境 を参照。
  • LAN内でも遅い → サーバーPCを有線LANに接続してみる。
  • ページをめくるたびに待つ → アプリのプリロード枚数を増やす（アプリ内設定）。
  • サーバーPC自体の負荷が高くないか確認する（他の重い処理が動いていないか）。

5. 接続しにくい環境について

インターネット接続の種類によっては、外出先からのHTTP直結ができない場合があります。

環境・回線の種類HTTP直結WebRTC説明
一般的な光回線（PPPoE） ○ ○ グローバルIPv4が割り当てられる。UPnPが有効なら最適な接続環境。
v6プラス / transix / MAP-E ○ ※ ○ IPv4アドレスを複数世帯で共有するが、ArcHiveはこの環境に対応済み。使えるポートを自動検出して開放します。
IPv6オプション（IPoE） ○ ○ グローバルIPv6が使える場合はIPv6直結も可能。
CGNAT（キャリアグレードNAT） ✕ ○ プロバイダー側でNATされるためポート開放不可。WebRTCのみ使用可能。
モバイルWi-Fiルーター / テザリング ✕ ○ モバイル回線はポート開放不可。外出先からはWebRTCのみ。
会社・学校のネットワーク ✕ △ セキュリティポリシーで多くのポートが制限されている。WebRTCも使えない場合あり。
v6プラス / transix / MAP-E をお使いの方へ：
「v6プラス」「transix」「OCNバーチャルコネクト」「v6コネクト」などはIPv4アドレスを複数世帯で共有する方式です。使えるポート番号に制限がありますが、ArcHive Serverはこの環境に対応しており、利用可能なポートを自動的に探して接続します。ログに MAP-E と表示された場合もこれが正常動作です。
WebRTCのみになってしまう場合：
CGNAT・モバイル回線などのポート開放ができない環境では、インターネット経由のP2P接続（WebRTC）のみ使用されます。機能はしますが、画像の読み込みが若干遅くなる場合があります。この状態ではサーバーのログに [WebRTC] DC接続 のみが表示されます。

6. よくある質問

Q. 「端末管理」に承認待ちが何件もたまっている

同じスマホでもアプリを再インストールすると新しい端末として登録されます。古い「承認待ち」や使わなくなった端末は「端末管理」から削除して構いません。

Q. 外出先ではつながるのに自宅に戻るとつながらなくなった

アプリがまだ「外出先モード（WebRTC等）」で接続しようとしている可能性があります。アプリを一度終了して再起動するか、接続先をリセットしてから「LAN内を探す」でもう一度ペアリングしてください。

Q. Windowsのファイアウォール設定はどうすればいい？

初回起動時にWindowsから「ネットワークアクセスを許可しますか？」というダイアログが表示されたら「アクセスを許可する」を選択してください。ダイアログを見逃した場合は、Windowsの「セキュリティ」→「ファイアウォールとネットワーク保護」→「アプリをファイアウォール経由で許可する」から ArcHiveServer（またはPythonで実行中の場合は Python）を許可してください。

Q. サーバーを閉じると外出先から接続できなくなる

サーバーを終了すると接続できなくなります。「ウィンドウ操作」の「閉じるボタン」を「システムトレイに格納」に設定すると、ウィンドウを閉じてもサーバーはバックグラウンドで動き続けます。タスクバーのトレイアイコンから再表示できます。

Q. 「ブラウザで開く」ボタンが押せない（グレーになっている）

「▶ サーバー起動」を押してからサーバーが起動するまで数秒かかります。ステータスが「起動中」に変わってからボタンが有効になります。

Q. ログに「UPnP: 失敗」と出る

ルーターのUPnP機能が無効になっているか、UPnPに対応していない可能性があります。ルーターの管理画面でUPnPを有効にしてみてください。有効にできない場合でも、WebRTC経由で外出先から接続することは可能です。

Q. PCを再起動したらつながらなくなった

PCを再起動するとサーバーも停止します。サーバーを再度起動してください。Windowsのスタートアップにサーバーを登録しておくと、PC起動時に自動でサーバーが起動します。
